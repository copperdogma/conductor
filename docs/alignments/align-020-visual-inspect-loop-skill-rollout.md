# Alignment 020 - Visual Inspect Loop Skill Rollout

Date: 2026-04-29

## Focus

Decide where doc-web's new `/visual-inspect-loop` skill should roll out across
the tracked repos.

Source skill:

`/Users/cam/Documents/Projects/doc-web/.agents/skills/visual-inspect-loop/SKILL.md`

The skill applies when correctness depends on visual, spatial, rendered, or
source-faithfulness judgment: screenshots, page images, crops, generated HTML or
PDF, slide/document renders, charts, OCR/layout output, UI renders, and image
goldens. Its important behavior is not just "run visual tests." It requires the
agent to personally inspect source and result artifacts, record concrete
quality gaps, generalize those gaps into reusable failure classes, patch the
pipeline, rerun, and reinspect before claiming quality.

## Classification

Portable improvement with selective sync.

## Decision

Roll the skill into repos that have current or clearly owned visual-artifact
surfaces. Do not blanket-sync it into repos whose current proof surfaces are
text/entity-only.

## Per-Repo Recommendation

| Repo | Recommendation | Reason |
| --- | --- | --- |
| doc-web | Source already has it | The repo owns scanned PDFs, page images, OCR/layout, crops, generated HTML, visual run artifacts, and image-backed eval/golden workflows. The user named this copy as the source. |
| Storybook | Sync now | Storybook owns UI/browser surfaces, photo and PDF artifact intake, media previews, debug timelines with image previews, sharing previews, and future video/photo workflows. |
| CineForge | Sync now | CineForge owns film, storyboard, visual-reference, image/video generation, previz, UI, and rendered media verification surfaces where tests alone do not prove quality. |
| Board Game Ingester | Sync now | Board Game Ingester owns scanned rulebooks, rules HTML, image asset extraction, crop bounds, contact sheets, visual crop-quality scoring, and image goldens. |
| Robo Rally | Sync now | Robo Rally is headless-first today, but it owns board/replay/client boundaries, future visual clients, and eventual boardgame-ingester asset intake; the skill should be ready for visual client and asset/golden work without moving game truth into the renderer. |
| Echo Forge | Sync now, UI/artifact use only | Echo Forge is audio-first, but it has product UI, webapp-testing, UI-scout, and generated artifact surfaces. Use this skill for UI/rendered/artifact inspection, not audio-quality review. |
| Dossier | Defer | Current evidence shows text/entity extraction, graph goldens, and eval classification rather than visual/image/rendered-document artifact ownership. If Dossier later owns a visual document lane directly, adopt then. |
| Conductor | Do not sync | Conductor records and supervises the rollout. It should not become the canonical skill source or claim product visual-artifact ownership. |
