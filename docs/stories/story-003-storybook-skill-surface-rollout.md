# Story 003 — Roll Out Storybook Skill Surface Alignment

**Priority**: High
**Status**: Done
**Decision Refs**: ADR-001, ADR-002
**Depends On**: None

## Goal

Turn Alignment 015 into execution-ready target-project work. Conductor should
carry Storybook's recent methodology skill improvements into the other tracked
apps with local adaptation instead of blanket text sync.

The source improvements are:

- `init-project` owns greenfield idea intake before `/setup-methodology`
- `/triage` runs a completion-sanity check before recommending maintenance or
  "nothing ready"
- setup, triage, eval, story, and build skills preserve the root/parent/child
  eval ladder before creating implementation backlog

## Acceptance Criteria

- [x] Conductor prepares isolated target-project worktrees or otherwise proves
      the target primary checkout is safe before editing any tracked project.
- [x] Echo Forge receives the Storybook eval-ladder guidance while preserving
      its Echo Forge-native selective-import language.
- [x] CineForge receives completion-sanity triage and eval-ladder guidance, and
      its older `init-project` surface is reconciled with the intake-first
      boundary without losing CineForge-specific benchmark, provider, UI-scout,
      or film-pipeline guidance.
- [x] Dossier receives completion-sanity and eval-ladder guidance, with an
      explicit decision about whether `init-project` is needed or whether
      `retrofit-ideal` remains the local intake equivalent.
- [x] Doc Web receives completion-sanity and eval-ladder guidance, with an
      explicit decision about whether to add a triage runbook and/or
      `init-project`.
- [x] Board Game Ingester receives the adapted greenfield intake/setup,
      completion-sanity, and eval-ladder package while preserving board-game
      benchmark, rulebook, component, and golden-fixture language.
- [x] Each changed target repo regenerates/checks cross-CLI wrappers and runs
      its repo-native methodology validation before Conductor marks that target
      aligned.
- [x] Conductor updates Alignment 015 or follow-up memory with the final target
      outcomes and any intentionally deferred sync.

## Out of Scope

- Editing target project primary checkouts that are dirty or behind when an
  isolated worktree is available
- Forcing exact Storybook wording into locally adapted skills
- Claiming sync is complete before target repos have their own validated
  patches, stories, or applied changes
- Broad registry/scout/build-map drift outside the Storybook skill-surface
  changes
